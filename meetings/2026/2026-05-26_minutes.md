### Charter Reviews

#### Browser Testing and Tools

* Simone Onofri: This is the group that does WebDriver, which is interesting and powerful, but also has many security implications. Testing relies on it. UI alerts are important for automation and for providing some safety. It is a tool for testing, not for general use. The question is whether we should add different considerations, or how we deal with the fact that useful tools can also be abused. The request is for charter review, not specification review.

* Tommaso Innocenti: What tools are they using behind that? Do Selenium and Puppeteer cover everything?

* Simone Onofri: There are different approaches for browser automation. For example, OpenAI has a tool where the internals are not clear, but it seems to use an extension to drive the browser. I am not sure whether WebDriver is used underneath.

* Charles Nevile: I can pick up the review, and someone else can also do another review. I think it is important to decide whether we are providing outdated feedback or retrospective feedback, or whether we should close the issues.

* Action: Charles Nevile to review the WebDriver charter for the 9 June agenda.

* Tommaso Innocenti: We can use different approaches. We are trying to provide help, even if this sometimes becomes a bottleneck. We are now managing the backlog.

* Action: Simone Onofri to review the WebDriver charter for the 9 June agenda.

#### WebMCP

* Simone Onofri: WebMCP wants wide review and plans to submit what they already did as self-review. They are not ready to send a threat model. They should also have privacy review. Tom has raised privacy concerns, not only security concerns. There is an opportunity to say something to them and invite them to look at the threat model in their security considerations. We need someone to read this so that we understand how it matches up.

* Tom Jones: The problem is that there is not a specification on how a browser should interact with MCP.

* Charles Nevile: Maybe this should be part of the specification.

* Tom Jones: I am confused about how the browser sends data.

* Giovanni Corti: There are discussions about telemetry. It is a delicate subject because it depends on the main purpose. I should check whether it is already classified as an asset.

* Simone Onofri: We also have concerns from Member States about telemetry. We should decide how to address this.

* Tommaso Innocenti: If we have AI in the browser, the AI needs context. It is hard to reason about the information being shared.

* Tom Jones: Should the browser be a threat actor? The question is how much data is shared for the sake of AI, and whether this is a threat to the user. It is connected to least privilege, especially with AI systems that grab everything.

* Giovanni Corti: I am also thinking about agentic AI. CRA will not address that here, but we can address it in W3C.

* Tom Jones: Should we add agentic AI to Threat Model for the Web?
